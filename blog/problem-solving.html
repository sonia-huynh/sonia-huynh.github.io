Problem Solving

A time I was blocked on a simple problem:

I was trying to compare whether key equaled studentName so I put (key = studentName), however this didn’t work because one equal sign is used to assign something. To check for a comparison I needed to use “==” in order for it to be an equality comparison operation.

A few techniques I used to try solve this problem was saying the code out loud to myself and explaining what I was trying to achieve. I also changed my code around to see if anything would work but I couldn't figure out the issue, and so I went onto the tech discord channel and found out someone else had a similar issue and once I saw it I realised what the fix was.

Initially I was confused and frustrated why my code wasn’t working. I was so sure that I did everything correctly and that it made sense to use an equal sign, but what I didn’t realise was that the number of equal signs have different functions.

Overall, through this, I learnt that it’s good to take a step back, take a look at some other people’s code and then learn from them. Because once I realised the differences between the equal signs, while I was completing other challenges and needed to use an equality comparison again, I knew straight away to use 2 equal signs and not fall in the trap of using just 1.

A time I elegantly solved a problem:

This time the challenge was to create a piece of code that would average a student’s grades for whichever student’s name was given to the code.

I found this challenge to be very similar to the one I previously did which was to add in a new score for a specific student. Because it was so similar I used the Rubber ducky method where I wrote out my code and then explained each piece of code out loud to myself and then console.logged to see if my code worked.

Throughout this process I was more calm and collected when problem solving, I felt like I understood what needed to be solved and how to solve it and my process worked. It was also because I was able to reflect on the previous code that I wrote that I was able to solve this one by myself and I was pretty proud of that because it felt like I was able to use what I learnt and put it into the correct context.

I learnt that more practice will make problem solving easier, I was able to solve this piece of code quicker than the rest because I understood what it did in the previous question and so I was able to use it in the correct context to solve this question. I also learnt that writing out code and explaining it out loud to myself was helpful in terms of getting me to understand what I was writing and why I wrote it. Lastly, I learnt that after completing the code and seeing that it works, I should go over it again to understand each component so that I know how it works and how I can use it again for next time in a different context.

Reflecting on my Problem Solving Techniques:

Pseudocoding: Moderately Confident

I tried to pseudocode for the code that I thought required a few lines and more thinking, howeverm this was harder than I thought because a few times after pseudocoding I realised I didn't even need to do those steps or it wasn't on the right track. So I ended up just skipping the pseudocoding and going straight into trying something.

Trying Something: Very Confident

I feel like my go to strategy to trying to solve a problem is just striaght up trying something, anything. As long as I have written some code, it is no longer a blank slate and I feel more confident in being able to solve the problem. So, I use trying something, very often.

Rubber Ducky Method: Confident

Another one I use often, although, sometimes I don't know if I'm even explaining the code correctly. But, saying everything out loud, helps me process the situation and I do end up feeling more confident that I can solve my problem.

Reading Error Messages: Somewhat Confident

I am confident I can read the error message, but I don't know what to do with the information afterwards sometimes haha. Unfortunately a lot of my error messages were also just "underfined" or the code was so terrible that I didn't even get an error message because the test couldn't run. But I will try my best to improve this.

Console Loggin: Somewhat Confident

I didn't console.log at the beginning because I was not used to it and didn't know what I was console logging, but near the end of the challenges I was using console log for each line of code and was feeling a bit more confident in how to use.

Googling: Confident

I'll count YouTubing and asking chatGPT as Googling as well. I did a looot of this. Nearly every challenge I was just googling, and watching youtube videos of people explaining new concepts to me. I used a lot of chatGPT to explain code to me in simpler ways and also how to use it in specific ways, so I can say I'm pretty confident in Googling.

Asking your peers for help: Not Confident

I think because everyone is online and I don't know who anyone is, it's hard for me to put myself out there and just ask people for help. I also feel like all my questions are stupid and I have a lot of questions so, I'm not very confident in asking my peers for help. However, I think this will improve during bootcamp as I get to meet everyone, I feel that I will be more confident in asking for help.

Asking coaches for help: A little Confident

I did ask a coach for help this time round so I'll put it as a little confident. I think what's holding me back is that I am stuck on every single challenge so I don't want to ask for help for every question and I want to try solve it on my own. I find that if I ask for help from the coaches they may end up just giving me the answer and I don't find that as satisfying as trying to figure it out on my own. However, I do believe that I should use the opportunity to ask my coaches questions.

Improving your process with reflection: Somewhat Confident

I'm confident in doing this because I think I'm constantly looking back at how I tried to solve a previous question and what I did during that time and how I can improve the way I'm attempting a new question.
