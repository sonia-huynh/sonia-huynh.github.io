Best Practices Associated with Using Classes Vs. Ids

What are Classes and Ids?

Classes and Ids are both attributes used to change elements. Ids are unique and specific to the element and can only be used for that element whereas Classes are not specific and can be used multiple times to target different elements. Think of Ids like a student id where each student has an id specific for them whereas Class is not specific and many different people can take the same class.

ID Class
Unique Non-specific
Specific


Let's look at some examples of how to use Classes and Ids:

Here, I have an example of a grocery list where I used Classes to categorize items in different colours based on food type as seen by fruit, dairy, meat and vegetables.




Let's say the supermarket is having a sale for Apples, Milk and Lettuce (yay!), I can use a new Class (class="sale") on the same grocery list to highlight in pink which products are on sale.

Note: To use two Classes together, you can put them in the same Class attribute " " with a space in between as seen in the example done for Apple, Milk and lettuce.






After hearing about the sale, we also find out that specifically Milk is having a buy 1 get 1 free deal! As the buy 1 get 1 free deal only applies to milk and not any of the other products, I can then use an Id attribute and name it "oneforone" to highlight Milk in yellow with a red dotted border around it to emphasise that only Milk has the special deal.

Here, we can see that using the Id attribute of "oneforone", it had overridden the both the "dairy" and "sale" Class style formatting that was on Milk.







So, what are the best practices of using Classes and Ids?

  • Use a Class when you want to apply the same styling to multiple different elements on the same page - good for reusability
  • Use Class to style blocks of content that you want to have a similar format - like I did with my example
  • Ids are unique within your page so they are useful if you want to style a specific section of your site without wanting to reuse the same styling somewhere else
  • Ids are also good for creating anchors within a page, just like what I did with this page, go on, click the "Move back up" in the footer to see it in action!
  • You can use Classes and Ids both together, but Id will always override the class so, use Id if you want to override other styles
  • You can use multiple classes together but you cannot do the same with an Id, remember, Id is unique


Recap on the best practices of using Classes and Ids:

  • Classes and Id differ because Id is specific and can only be associated with one element whereas classes can be used for multiple different elements.
  • You can use Classes and Ids both together, but Id will always override the Class
  • Classes are resusable so they are useful if you want to use the same styling requirements for multiple elements on your site
  • You can use multiple Classes together but you cannot do the same with an Id
  • Ids are unique within your page so they are useful if you want to use it for a specific section only
