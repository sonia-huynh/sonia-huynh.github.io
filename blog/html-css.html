The Best Practices Associated with Using Classes Vs. Ids

What are Classes and Ids?

Classes and Ids are both attributes used to change elements. Ids are unique and specific to the element and can only be used for that element whereas classes are not specific and can be used multiple times to target different elements. Think of ids like a student id where each student has an id specific for them whereas class is not specific and many different people can take the same class.

Unique Non-specific

Here, I have an example of a grocery list where I used classes to categorize items based on fruit, dairy, meat and vegetables in different colours.




Let's say the supermarket is having a dairy product sale (yay!), I can use a new class (class=sale) on the same list to highlight in pink which products have a sale (Milk and Cheese).

To use two classes together, you can put them in the same " " with a space in between as seen in the example done for Milk and Cheese.






After hearing about the dairy product sale, we also find out that specifically Milk is having a buy 1 get 1 free offer. As the buy 1 get 1 free offer only applies to milk and not all dairy products, I can then use an ID attribute to highlight Milk in yellow with a red dotted border around it to emphasise that only Milk has the special deal.

Here, we can see that using the ID attribute of "oneforone", it had overridden the both the "dairy" and "sale" class formatting that was on Milk.
